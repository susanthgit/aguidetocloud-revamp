## What You'll Learn

This full AZ-204 course prepares you for the **Microsoft Azure Developer Associate** certification exam. You'll learn how to:

- **Build and deploy web apps** using Azure App Service and Azure Functions
- **Implement containerised solutions** with Azure Container Instances and Azure Container Apps
- **Work with Azure storage** — Blob Storage, Cosmos DB, and Azure Cache for Redis
- **Secure your applications** using Microsoft Entra ID, Managed Identities, and Key Vault
- **Monitor and troubleshoot** with Application Insights and Azure Monitor
- **Implement API Management** and event-driven solutions with Event Grid and Service Bus

> **Tip:** This is an associate-level exam — some programming experience is expected. Follow along with the demos and try building the solutions in your own Azure subscription.

## Key Topics Covered

### Azure Compute Solutions (25-30%)

The largest section of the exam focuses on building and deploying applications:

- **Azure App Service** — The go-to platform for hosting web apps, REST APIs, and mobile backends. Supports .NET, Java, Node.js, Python, and PHP.
- **Azure Functions** — Serverless compute that runs code on demand. Perfect for event-driven architectures — you pay only when your code executes.
- **Azure Container Instances (ACI)** — Run Docker containers without managing servers. Great for quick deployments, batch jobs, and dev/test scenarios.
- **Azure Container Apps** — Managed Kubernetes-based platform for running microservices and containerised apps at scale.

### Azure Storage Solutions (15-20%)

- **Azure Blob Storage** — Store unstructured data (images, documents, backups). Understand hot, cool, and archive access tiers.
- **Azure Cosmos DB** — Globally distributed, multi-model database with single-digit millisecond response times. Supports SQL, MongoDB, Cassandra, and Gremlin APIs.
- **Azure Cache for Redis** — In-memory data store for caching, session management, and real-time analytics.

### Security and Authentication (20-25%)

- **Microsoft Entra ID integration** — Implement authentication using Microsoft Authentication Library (MSAL) and OAuth 2.0 / OpenID Connect.
- **Managed Identities** — Let Azure resources authenticate to other services without storing credentials in code.
- **Azure Key Vault** — Securely store and manage secrets, certificates, and encryption keys.
- **Azure App Configuration** — Centralised configuration management for your applications.

### Monitoring and Logging (15-20%)

- **Application Insights** — Track application performance, failures, and user behaviour.
- **Azure Monitor** — Collect, analyse, and act on telemetry from your Azure resources.
- **Log Analytics** — Query logs using KQL (Kusto Query Language) to diagnose issues.

### API Management and Messaging (15-20%)

- **Azure API Management** — Publish, secure, and manage APIs at scale.
- **Azure Event Grid** — Event-driven messaging for reactive architectures.
- **Azure Service Bus** — Enterprise messaging for reliable, ordered message delivery.
- **Azure Queue Storage** — Simple queue service for decoupling application components.

## Frequently Asked Questions

**Is the AZ-204 exam hard?**
It's an associate-level exam, so it requires real development knowledge. If you have 1-2 years of programming experience and study the Azure-specific services, you can pass with focused preparation.

**Do I need to know a specific programming language?**
The exam is language-agnostic, but most examples use C# or Python. You should be comfortable reading code in at least one language.

**Should I take AZ-900 before AZ-204?**
AZ-900 is recommended but not required. If you already have cloud or Azure experience, you can go directly to AZ-204.

**What's the difference between AZ-204 and AZ-400?**
AZ-204 focuses on **developing** applications for Azure. AZ-400 focuses on **DevOps practices** — CI/CD, infrastructure as code, and monitoring. Many developers pursue both.

**How long should I study for AZ-204?**
Plan for 4-6 weeks of focused study (2-3 hours per day) including hands-on labs. Developers with existing Azure experience may need less time.
