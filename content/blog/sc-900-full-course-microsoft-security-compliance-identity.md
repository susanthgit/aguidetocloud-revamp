## Key Topics Covered

### Security Concepts

The exam starts with foundational security concepts that apply across all cloud platforms:

- **Zero Trust model** — "Never trust, always verify." Every request is authenticated and authorised regardless of where it comes from.
- **Shared responsibility model** — In cloud computing, security responsibilities are split between Microsoft (infrastructure) and you (data, identities, applications).
- **Defence in depth** — Multiple layers of security controls (identity, network, compute, data) so that if one layer fails, others still protect you.
- **Common threats** — Phishing, ransomware, identity attacks, DDoS, and insider threats.

### Microsoft Entra ID (Azure AD)

Microsoft Entra ID is the cloud-based identity and access management service at the heart of Microsoft 365 and Azure:

- **Authentication vs authorisation** — Authentication proves who you are; authorisation determines what you can access
- **Single sign-on (SSO)** — One set of credentials for all your apps
- **Multi-factor authentication (MFA)** — Something you know + something you have + something you are
- **Conditional access** — Policies that enforce security requirements based on signals (location, device, risk level)
- **B2B and B2C** — Collaborating with external partners (B2B) and building customer-facing apps (B2C)

### Microsoft Security Solutions

- **Microsoft Defender for Cloud** — Cloud security posture management (CSPM) and workload protection
- **Microsoft Sentinel** — Cloud-native SIEM and SOAR for threat detection and response
- **Microsoft Defender XDR** — Extended detection and response across endpoints, email, identity, and cloud apps
- **Microsoft Defender for Endpoint** — Endpoint protection for Windows, macOS, Linux, iOS, and Android

### Compliance Solutions

- **Microsoft Purview** — Unified compliance and data governance platform
- **Information protection** — Sensitivity labels, encryption, and data classification
- **Data loss prevention (DLP)** — Prevent accidental sharing of sensitive data
- **Insider risk management** — Detect and respond to risky activities within your organisation
- **eDiscovery** — Legal search and hold for compliance investigations
- **Audit logs** — Track user and admin activities across Microsoft 365

## Frequently Asked Questions

**Is the SC-900 exam difficult?**
No — it's a fundamentals-level exam designed for beginners. If you watch the full course and review Microsoft Learn materials, you should be well-prepared.

**Do I need security experience for SC-900?**
No prior security experience is needed. The exam covers concepts and awareness, not deep technical configuration.

**What certification does SC-900 lead to?**
SC-900 is a great stepping stone to SC-200 (Security Operations Analyst), SC-300 (Identity and Access Administrator), and SC-400 (Information Protection Administrator).

**Is there a free way to take the SC-900?**
Microsoft occasionally offers free exam vouchers through [Microsoft Virtual Training Days](https://events.microsoft.com/en-us/mvtd) and the [Cloud Skills Challenge](https://learn.microsoft.com/en-us/training/cloud-skills-challenge/).

**How long should I study for SC-900?**
Most people need 1-2 weeks of focused study (2-3 hours per day) if starting from scratch.
