## What You'll Learn

This complete AZ-400 course covers everything for the **Microsoft DevOps Engineer Expert** certification. You'll master:

- **CI/CD pipelines** — Build, test, and deploy applications automatically with Azure Pipelines
- **Source control management** — Git branching strategies, pull requests, and Azure Repos
- **Infrastructure as Code (IaC)** — Automate infrastructure deployment with ARM templates, Bicep, and Terraform
- **Continuous monitoring** — Application Insights, Azure Monitor, and log analytics
- **Security in DevOps** — Secret management, dependency scanning, and secure pipelines
- **DevOps culture and processes** — Agile planning, feedback loops, and collaboration practices

> **Tip:** This is an expert-level certification. Having hands-on experience with Azure DevOps or GitHub Actions will significantly help your preparation.

## Key Topics Covered

### Development Process Configuration (15-20%)

- **Azure DevOps vs GitHub** — choosing the right platform for your team
- **Azure Repos** — Git repositories, branching strategies (GitFlow, trunk-based development), and pull request workflows
- **Work item tracking** — Azure Boards for agile planning, sprints, and backlogs
- **Package management** — Azure Artifacts for NuGet, npm, Maven, and Python packages

### CI/CD Pipelines (40-45%)

This is the largest section — building automated build and release pipelines:

- **Azure Pipelines** — YAML-based pipelines for building, testing, and deploying applications
- **Build agents** — Microsoft-hosted vs self-hosted agents, scaling agent pools
- **Multi-stage pipelines** — Dev → Staging → Production with approval gates
- **Deployment strategies** — Blue-green, canary, rolling deployments, and feature flags
- **GitHub Actions** — CI/CD workflows using GitHub's built-in automation platform
- **Release management** — Approvals, gates, and deployment groups

### Infrastructure as Code (10-15%)

- **ARM templates** — JSON-based Azure resource deployment templates
- **Bicep** — Microsoft's domain-specific language for Azure deployments (simpler than ARM)
- **Terraform** — Multi-cloud infrastructure provisioning tool
- **Azure Resource Manager** — Understanding deployment scopes, resource groups, and subscriptions

### Security and Compliance (10-15%)

- **Secret management** — Azure Key Vault integration with pipelines
- **Dependency scanning** — Detect vulnerable packages before they reach production
- **Container security** — Scanning images, signed images, and secure registries
- **Compliance as code** — Azure Policy, governance, and compliance automation

### Monitoring and Feedback (10-15%)

- **Application Insights** — Real-time application performance monitoring
- **Azure Monitor** — Alerts, dashboards, and log queries with KQL
- **Site Reliability Engineering (SRE)** — Error budgets, SLIs, SLOs, and incident management
- **Feedback integration** — Feature flags, A/B testing, and user telemetry

## Study Tips for the AZ-400

1. **Build real pipelines** — create a sample project with CI/CD in Azure DevOps or GitHub Actions
2. **Practice IaC** — deploy resources using Bicep or Terraform
3. **Understand both Azure DevOps and GitHub** — the exam covers both platforms
4. **Complete the AZ-204 or AZ-104 first** — the AZ-400 requires one of these as a prerequisite
5. **Focus on YAML pipelines** — the exam emphasises YAML over classic editor

## Frequently Asked Questions

**Is AZ-400 harder than AZ-204?**
Yes — AZ-400 is expert-level and covers a broader scope including CI/CD, IaC, security, and monitoring. It expects practical experience with DevOps tooling.

**Do I need both AZ-104 and AZ-204 for AZ-400?**
No — you need either AZ-104 OR AZ-204 as a prerequisite, not both.

**Should I learn Azure DevOps or GitHub Actions?**
Both — the exam covers both platforms. Most organisations use one or the other, but understanding both gives you flexibility.

**How long should I study for AZ-400?**
Plan for 6-8 weeks with hands-on practice. If you're already using Azure DevOps daily, you may need less time.

**What career roles need the AZ-400?**
DevOps engineers, release managers, platform engineers, and SRE (Site Reliability Engineer) roles commonly require or prefer the AZ-400.
