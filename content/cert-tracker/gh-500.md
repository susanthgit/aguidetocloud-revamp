## About the GH-500 Exam

> Master GitHub Advanced Security (GHAS) — the GitHub security suites and ecosystem, Secret Protection with push protection, supply chain security with Dependabot and dependency review, Code Security with CodeQL and Copilot Autofix, security operations and campaigns, and enterprise security administration.

The complete practice exam for GitHub Advanced Security (GH-500). Covers the GitHub security suites and ecosystem (Code Security, Secret Protection, Supply Chain Security, Security Overview, and how a secure SDLC fits together), configuring and using Secret Protection (push protection, validity checks, custom secret patterns, delegated bypass policies, alert lifecycle), supply chain security (dependency graph, SBOM export, GitHub Advisory Database, Dependabot alerts and security updates, EPSS prioritization, dependency review, security campaigns), Code Security (CodeQL default and advanced setup, SARIF interoperability, build modes, matrix builds, Copilot Autofix, dataflow triage), security operations (CVE/CWE/GHSA concepts, severity and remediation rulesets, query-suite customization, cross-suite governance, shift-left prevention), and GitHub security suites administration (security configurations, GHEC vs GHES, enterprise and organization policies, roles and bypass permissions, automation APIs) — every question a real-world scenario with full explanations.

## Who Should Take This Exam?

The GH-500 is designed for **security engineers, application security specialists, DevSecOps engineers, and platform administrators** who roll out and operate GitHub Advanced Security at organization and enterprise scale. Hands-on experience securing code, secrets, and dependencies with GHAS is recommended.

**Prerequisites:** None (working knowledge of GitHub, CI/CD, and secure development recommended)

**Typical study time:** 4-8 weeks of focused study

## Exam Domains & Weights

The GH-500 exam covers **6 domains**. Focus your study time based on the weights below — higher-weighted domains have more exam questions.

| Domain | Weight | Practice Qs |
|--------|--------|-------------|
| Describe GitHub security suites, features, and ecosystem | 20% | 50 |
| Configure and use Secret Protection | 17% | 43 |
| Configure and use supply chain security | 17% | 42 |
| Configure and use Code Security | 15% | 38 |
| Security operations: best practices, prioritization, and remediation | 18% | 45 |
| GitHub security suites administration | 13% | 32 |
| **Total** | **100%** | **250** |

> 💡 **Study tip:** The ecosystem domain (20%) and security operations (18%) together carry well over a third of the exam, so make sure you can reason across all three suites — Code Security, Secret Protection, and Supply Chain Security — not just one. Secret Protection and supply chain security (17% each) reward hands-on practice with push protection, Dependabot, and dependency review. Administration (13%) is smaller but the security-configurations model is heavily tested — don't skip it.
