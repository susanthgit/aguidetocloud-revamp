## About the OCI-OBSERVABILITY-PROFESSIONAL Exam

> Master the Oracle Cloud Infrastructure 2026 Observability Professional exam (1Z0-1111-26) — monitoring OCI workloads end to end with the OCI Observability & Management suite: Monitoring with MQL and alarms, Notifications, Events, Connector Hub, Logging, Logging Analytics with Cluster and Link analysis, Application Performance Monitoring with distributed tracing, Real User Monitoring and Availability Monitoring, and Stack Monitoring.

The complete practice exam for the Oracle Cloud Infrastructure 2026 Observability Professional certification (exam 1Z0-1111-26, validated against Oracle Cloud Infrastructure 2026). A professional-level certification for site-reliability engineers, cloud operations engineers, DevOps engineers, and platform engineers who monitor, alert on, diagnose, and visualize OCI workloads using the OCI Observability & Management suite. Covers the Pillars of Observability (metrics, logs, traces and events and how the O&M services map to them), Monitoring Cloud Environments with Metrics and Alarms (metric namespaces and dimensions, the Monitoring Query Language MQL with its dot-suffix statistics and absence detection, Metrics Explorer, custom metrics with PostMetricData, metric-based alarms with trigger delay, severity and suppression, and Notifications topics and subscriptions), Responding to Cloud Resource Changes in Real-Time (OCI Events in CloudEvents format, event rules with condition and action, and the three Events actions Notifications, Streaming and Functions, plus the distinctions between Events, Streaming, Queue and Notifications), Centrally Managing and Visualizing Log Data (the three OCI Logging categories Audit, Service and Custom, log groups, the Unified Monitoring Agent, and Connector Hub source-to-target routing), Identifying Log Data Patterns and Creating Visualizations for Advanced Analytics (Oracle Logging Analytics entities, sources, parsers, labels and lookups, the Log Explorer, Cluster analysis for grouping similar records and Link analysis for grouping records over time, and dashboards), Monitoring Applications with Deep Visibility into End-User Experience (OCI Application Performance Monitoring domains and public versus private data keys, distributed tracing with spans and the Trace Explorer, Real User Monitoring, Server Monitoring, and Availability Monitoring formerly Synthetic Monitoring with scripted monitors and public and dedicated vantage points), and Monitoring Distributed Components of an Application Stack (OCI Stack Monitoring with the Management Agent, resource discovery, Enterprise Summary and Stack View topology, metric extensions and alarm integration) — every question a real-world scenario with full explanations, hints, exam tips, and why-wrong rationales.

> 💡 **Study tip:** **Identify Log Data Patterns and Create Visualizations for Advanced Analytics** (22%) and **Monitor Applications with Deep Visibility into End-User Experience** (20%) together are over 40% of the exam — start there. Get the confusable details right: OCI MQL is dot-suffix (`CpuUtilization[1m].mean()`, not a PromQL-style prefix); in Logging Analytics **Cluster** groups similar records into patterns while **Link** groups records by a field over time; APM's **Availability Monitoring** is the current name for the former Synthetic Monitoring, and the **public** data key is for browser/RUM while the **private** key stays server-side. Remember alarms are metric-based and drive automation through a Notifications topic with an Oracle Functions subscription — OCI Monitoring alarms do not emit OCI Events.
