## Who is this exam for?

The AZ-305 is Microsoft's expert-level Azure architecture exam. Unlike the AZ-104 (which tests hands-on administration), the AZ-305 tests your ability to **design** solutions — choosing the right services, patterns, and trade-offs for given business requirements.

You should have advanced experience in Azure administration, networking, security, data platforms, and governance. The exam uses case studies and design scenarios where you recommend solutions based on requirements like cost, performance, security, and reliability. Think of it as "which Azure service should I use and why?"

**💡 Tip:** Study the [Azure Well-Architected Framework](https://learn.microsoft.com/en-us/azure/well-architected/) and the [Cloud Adoption Framework](https://learn.microsoft.com/en-us/azure/cloud-adoption-framework/) — both are heavily referenced.

### Design business continuity solutions (15–20%)

Business continuity is about keeping things running when something goes wrong. This domain covers designing backup strategies (what to back up, how often, where to store it), disaster recovery (replicating workloads to a secondary region), and high availability (ensuring services survive individual component failures).

#### Design solutions for backup and disaster recovery

Know the difference between RPO (how much data you can afford to lose) and RTO (how quickly you need to recover). Different workloads need different strategies — VMs use Azure Backup and Site Recovery, databases have built-in geo-replication, and unstructured data uses storage redundancy.

- [Recommend a recovery solution for Azure and hybrid workloads that meets recovery objectives](https://learn.microsoft.com/en-us/training/modules/design-solution-for-backup-disaster-recovery/2-design-for-backup-recovery)
- [Recommend a backup and recovery solution for compute](https://learn.microsoft.com/en-us/training/modules/design-solution-for-backup-disaster-recovery/3-design-for-azure-backup)
- [Recommend a backup and recovery solution for databases](https://learn.microsoft.com/en-us/training/modules/design-solution-for-backup-disaster-recovery/4-design-for-azure-blob-backup)
- [Recommend a backup and recovery solution for unstructured data](https://learn.microsoft.com/en-us/training/modules/design-solution-for-backup-disaster-recovery/5-design-for-azure-site-recovery)

### Design infrastructure solutions (30–35%)

This is the largest domain — over a third of the exam. It covers designing compute solutions (VMs, containers, serverless), application architectures (messaging, events, APIs, caching), migration strategies, and network designs. Expect complex case study questions here.

#### Design compute solutions

You need to match the right compute service to the workload requirements. VMs for lift-and-shift, containers for microservices, App Service for web apps, Functions for event-driven workloads, and Batch for large-scale parallel processing.

- [Specify components of a compute solution based on workload requirements](https://learn.microsoft.com/en-us/training/modules/design-compute-solution/2-choose-compute-service)
- [Recommend a virtual machine-based solution](https://learn.microsoft.com/en-us/training/modules/design-compute-solution/3-design-for-azure-virtual-machine-solutions)
- [Recommend a container-based solution](https://learn.microsoft.com/en-us/training/modules/design-compute-solution/4-design-for-azure-container-solutions)
- [Recommend a serverless-based solution](https://learn.microsoft.com/en-us/training/modules/design-compute-solution/5-design-for-azure-functions-solutions)
- [Recommend a compute solution for batch processing](https://learn.microsoft.com/en-us/training/modules/design-compute-solution/6-design-for-azure-logic-app-solutions)

#### Design an application architecture

Modern applications are built from loosely coupled components that communicate through messages and events. You need to know when to use Service Bus vs Event Grid vs Event Hubs, when to add a caching layer (Redis), and how to design API integrations.

- [Recommend a messaging architecture](https://learn.microsoft.com/en-us/training/modules/design-application-architecture/2-describe-message-queue-solutions)
- [Recommend an event-driven architecture](https://learn.microsoft.com/en-us/training/modules/design-application-architecture/3-describe-event-driven-architecture)
- [Recommend a solution for API integration](https://learn.microsoft.com/en-us/training/modules/design-application-architecture/4-describe-api-integration)
- [Recommend a caching solution for applications](https://learn.microsoft.com/en-us/training/modules/design-application-architecture/5-describe-caching-solution)
- [Recommend an application configuration management solution](https://learn.microsoft.com/en-us/training/modules/design-application-architecture/6-describe-app-config)
- [Recommend an automated deployment solution for applications](https://learn.microsoft.com/en-us/training/modules/design-application-architecture/7-describe-automated-deployment)

#### Design migrations

Migration is a major topic — moving on-premises workloads to Azure. You need to know the Cloud Adoption Framework, assessment tools (Azure Migrate), and when to recommend IaaS (lift-and-shift) vs PaaS (modernise). Database migrations and unstructured data migrations have their own patterns.

- [Evaluate a migration solution that leverages the Cloud Adoption Framework for Azure](https://learn.microsoft.com/en-us/training/modules/design-migrations/2-evaluate-migration-with-cloud-adoption-framework)
- [Evaluate on-premises servers, data, and applications for migration](https://learn.microsoft.com/en-us/training/modules/design-migrations/3-assess-with-azure-migrate)
- [Recommend a solution for migrating workloads to IaaS and PaaS](https://learn.microsoft.com/en-us/training/modules/design-migrations/4-select-migration-tool)
- [Recommend a solution for migrating databases](https://learn.microsoft.com/en-us/training/modules/design-migrations/5-migrate-databases)
- [Recommend a solution for migrating unstructured data](https://learn.microsoft.com/en-us/training/modules/design-migrations/6-migrate-unstructured-data)

#### Design network solutions

Networking ties everything together. You need to design connectivity to the internet (public IPs, NAT Gateway), to on-premises (VPN Gateway, ExpressRoute), optimise performance (Front Door, CDN, Traffic Manager), secure the network (firewalls, NSGs, private endpoints), and balance loads.

- [Recommend a connectivity solution that connects Azure resources to the internet](https://learn.microsoft.com/en-us/training/modules/design-network-solutions/2-design-for-on-premises-connectivity-to-azure)
- [Recommend a connectivity solution that connects Azure resources to on-premises networks](https://learn.microsoft.com/en-us/training/modules/design-network-solutions/3-design-for-azure-network-connectivity-services)
- [Recommend a solution to optimize network performance](https://learn.microsoft.com/en-us/training/modules/design-network-solutions/4-design-for-application-delivery-services)
- [Recommend a solution to optimize network security](https://learn.microsoft.com/en-us/training/modules/design-network-solutions/5-design-for-application-protection-services)
- [Recommend a load-balancing and routing solution](https://learn.microsoft.com/en-us/training/modules/design-network-solutions/4-design-for-application-delivery-services)
