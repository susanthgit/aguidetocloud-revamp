## Who is this exam for?

The SC-200 is for security operations analysts — the people who monitor, investigate, and respond to security threats across an organisation's multi-cloud and on-premises environment. Your daily tools are Microsoft Defender XDR, Microsoft Sentinel, and KQL (Kusto Query Language).

This exam was **significantly updated on April 16, 2026** with major restructuring — several domains were merged, "Copilot for Security" became embedded throughout (rather than a standalone section), and the threat hunting domain was expanded. Make sure your study materials are current.

**💡 Tip:** KQL is essential for this exam. If you're not comfortable writing KQL queries, start with the [KQL learning path](https://learn.microsoft.com/en-us/training/paths/sc-200-utilize-kql-for-azure-sentinel/) before diving into the other content.

## Skills Measured — with Microsoft Learn Links

### Manage a security operations environment (40–45%)

This is the largest domain — nearly half the exam. It covers configuring the Defender XDR and Sentinel platforms, ingesting data from various sources, and creating detection rules. You need to know how to set up the SOC environment, not just respond to alerts.

#### Configure automation for Microsoft Defender XDR and Microsoft Sentinel

This sub-area covers configuring notifications, alert tuning, automated investigation and response (AIR), attack disruption, and Sentinel automation rules and playbooks. It's about making your SOC more efficient through automation.

- [Configure email notifications in Microsoft Defender XDR](https://learn.microsoft.com/en-us/defender-xdr/configure-email-notifications)
- [Configure alert notifications in Microsoft Defender XDR, including tuning, suppression, and correlation](https://learn.microsoft.com/en-us/defender-xdr/investigate-alerts)
- [Configure Microsoft Defender for Endpoint advanced features](https://learn.microsoft.com/en-us/defender-endpoint/advanced-features)
- [Configure rules settings in Microsoft Defender for Endpoint](https://learn.microsoft.com/en-us/defender-endpoint/configure-endpoints)
- [Configure custom data collection in Microsoft Defender for Endpoint](https://learn.microsoft.com/en-us/defender-endpoint/data-collection-analyzer)
- [Configure security policies for Defender for Endpoint, including ASR rules](https://learn.microsoft.com/en-us/defender-endpoint/attack-surface-reduction)
- [Manage automated investigation and response capabilities in Microsoft Defender XDR](https://learn.microsoft.com/en-us/defender-xdr/m365d-autoir)
- [Configure automatic attack disruption in Microsoft Defender XDR](https://learn.microsoft.com/en-us/defender-xdr/automatic-attack-disruption)
- [Configure and manage device groups, permissions, and automation levels in Defender for Endpoint](https://learn.microsoft.com/en-us/defender-endpoint/machine-groups)
- [Create and configure automation rules in Microsoft Sentinel](https://learn.microsoft.com/en-us/azure/sentinel/automate-incident-handling-with-automation-rules)
- [Create and configure Microsoft Sentinel playbooks](https://learn.microsoft.com/en-us/azure/sentinel/tutorial-respond-threats-playbook)

#### Configure the Microsoft Sentinel SIEM and platform

Setting up Sentinel correctly is fundamental. You need to know the roles, data retention tiers (Analytics, Data Lake, XDR), workbook creation, and SOC optimisation recommendations.

- [Specify Microsoft Sentinel roles](https://learn.microsoft.com/en-us/azure/sentinel/roles)
- [Manage data retention for XDR and Microsoft Sentinel tables](https://learn.microsoft.com/en-us/azure/sentinel/data-retention)
- [Create and configure Microsoft Sentinel workbooks](https://learn.microsoft.com/en-us/azure/sentinel/monitor-your-data)
- [Optimise the Microsoft Sentinel platform, including SOC optimisation recommendations](https://learn.microsoft.com/en-us/azure/sentinel/soc-optimization/soc-optimization-access)

#### Ingest data into the Microsoft Sentinel SIEM and platform

Sentinel is only as good as the data it receives. This sub-area covers connecting data sources — Windows Security Events via AMA, Syslog/CEF connectors, Azure Activity logs, and custom log tables. You also need to know how to ingest threat intelligence.

- [Select data connectors based on data source requirements](https://learn.microsoft.com/en-us/azure/sentinel/connect-data-sources)
- [Configure collection of Windows Security events by using Windows Security Events via AMA](https://learn.microsoft.com/en-us/azure/sentinel/connect-services-windows-based)
- [Plan and configure collection of Windows Security events by using WEF](https://learn.microsoft.com/en-us/azure/sentinel/connect-services-windows-based)
- [Plan and configure Syslog via AMA and CEF via AMA connectors](https://learn.microsoft.com/en-us/azure/sentinel/connect-cef-syslog-options)
- [Configure collection of Azure activities by using Azure Policy and resource diagnostic settings](https://learn.microsoft.com/en-us/azure/sentinel/connect-azure-activity)
- [Ingest threat indicators into Microsoft Sentinel](https://learn.microsoft.com/en-us/azure/sentinel/understand-threat-intelligence)
- [Create custom log tables in the workspace to store ingested data](https://learn.microsoft.com/en-us/azure/sentinel/create-custom-connector)

#### Configure detections

Detection rules are how Sentinel identifies threats. Scheduled rules run KQL queries at regular intervals, NRT (near-real-time) rules detect threats within minutes, and machine learning rules use anomaly detection. You also need to map your detections to the MITRE ATT&CK framework.

- [Create custom detection rules by using Advanced Hunting in Microsoft Defender XDR](https://learn.microsoft.com/en-us/defender-xdr/custom-detections-overview)
- [Manage custom detection rules in Microsoft Defender XDR](https://learn.microsoft.com/en-us/defender-xdr/custom-detection-rules)
- [Configure and manage analytics rules in Microsoft Sentinel, including scheduled, NRT, threat intelligence, and ML](https://learn.microsoft.com/en-us/azure/sentinel/detect-threats-built-in)
- [Analyse attack vector coverage by using the MITRE ATT&CK matrix](https://learn.microsoft.com/en-us/azure/sentinel/mitre-coverage)
- [Configure anomalies in Microsoft Sentinel](https://learn.microsoft.com/en-us/azure/sentinel/soc-ml-anomalies)

### Respond to security incidents (35–40%)

This domain covers the core SOC workflow: an alert fires, you triage it, investigate, and respond. You need to investigate across multiple Defender products, handle complex multi-stage attacks, manage incidents with case management, and use Copilot for Security as an embedded investigation tool.

#### Respond to alerts and incidents in Microsoft Defender XDR

- [Investigate and remediate threats by using Defender for Office 365, including automatic attack disruption](https://learn.microsoft.com/en-us/defender-office-365/threat-explorer-real-time-detections-about)
- [Investigate and remediate threats or compromised entities identified by Microsoft Purview](https://learn.microsoft.com/en-us/purview/alert-policies)
- [Investigate and remediate alerts from Microsoft Defender for Cloud workload protections](https://learn.microsoft.com/en-us/defender-for-cloud/managing-and-responding-alerts)
- [Investigate and remediate security risks identified by Defender for Cloud Apps](https://learn.microsoft.com/en-us/defender-cloud-apps/investigate)
- [Investigate and remediate compromised identities identified by Microsoft Entra ID](https://learn.microsoft.com/en-us/entra/id-protection/howto-identity-protection-investigate-risk)
- [Investigate and remediate security alerts from Microsoft Defender for Identity](https://learn.microsoft.com/en-us/defender-for-identity/manage-security-alerts)
- [Investigate and remediate alerts and incidents identified by Microsoft Sentinel](https://learn.microsoft.com/en-us/azure/sentinel/investigate-cases)
- [Investigate incidents by using agentic AI, including embedded Copilot for Security](https://learn.microsoft.com/en-us/copilot/security/experiences/microsoft-defender-xdr-plugin)
- [Investigate complex attacks, such as multi-stage, multi-domain, and lateral movement](https://learn.microsoft.com/en-us/defender-xdr/incidents-overview)
- [Manage security incidents by using case management](https://learn.microsoft.com/en-us/defender-xdr/manage-incidents)

#### Respond to alerts and incidents in Microsoft Defender for Endpoint

When a device is compromised, you need to dig into the device timeline, run live response commands, collect investigation packages, and work with automatic attack disruption. This is very hands-on.

- [Investigate device timelines](https://learn.microsoft.com/en-us/defender-endpoint/device-timeline-event-flag)
- [Perform actions on the device, including live response and collecting investigation packages](https://learn.microsoft.com/en-us/defender-endpoint/live-response)
- [Perform evidence and entity investigation](https://learn.microsoft.com/en-us/defender-endpoint/investigate-files)
- [Investigate and remediate incidents identified by automatic attack disruption](https://learn.microsoft.com/en-us/defender-xdr/automatic-attack-disruption)

#### Investigate Microsoft 365 activities to identify threats

Beyond Defender alerts, you can use Microsoft Purview Audit, Content Search, and Microsoft Graph activity logs to investigate suspicious activity across the M365 tenant.

- [Investigate threats by using Audit from Microsoft Purview](https://learn.microsoft.com/en-us/purview/audit-search)
- [Investigate threats by using Content Search in Microsoft Purview](https://learn.microsoft.com/en-us/purview/ediscovery-content-search-overview)
- [Investigate threats by using Microsoft Graph activity logs](https://learn.microsoft.com/en-us/graph/microsoft-graph-activity-logs-overview)

### Perform threat hunting (20–25%)

Threat hunting is proactive — you go looking for threats that haven't triggered any alerts. KQL is your primary tool. This domain covers writing Advanced Hunting queries in Defender XDR and hunting queries in Sentinel, including the new Sentinel MCP Server integration and Data Lake queries.

#### Detect threats by using Microsoft Defender XDR

- [Identify the appropriate table to use in a KQL query](https://learn.microsoft.com/en-us/defender-xdr/advanced-hunting-schema-tables)
- [Identify threats by using Kusto Query Language (KQL)](https://learn.microsoft.com/en-us/training/modules/construct-kql-statements/)
- [Create Advanced Hunting queries](https://learn.microsoft.com/en-us/defender-xdr/advanced-hunting-query-language)
- [Interpret threat analytics in Microsoft Defender XDR](https://learn.microsoft.com/en-us/defender-xdr/threat-analytics)
- [Create hunting graphs, including blast radius](https://learn.microsoft.com/en-us/defender-xdr/advanced-hunting-overview)
- [Analyse relationships between entities by using Sentinel Graph](https://learn.microsoft.com/en-us/azure/sentinel/investigate-cases)

#### Detect threats by using the Microsoft Sentinel platform

- [Create and monitor hunting queries](https://learn.microsoft.com/en-us/azure/sentinel/hunting)
- [Create and manage KQL jobs in Data Lake](https://learn.microsoft.com/en-us/azure/sentinel/hunting)
- [Create and manage Summary rule tables for querying](https://learn.microsoft.com/en-us/azure/sentinel/summary-rules)
