## Who is this exam for?

The PL-300 is for Power BI data analysts — the people who turn raw data into meaningful visualisations and insights. You connect to data sources, clean and transform data with Power Query, build data models, write DAX formulas, create reports, and manage Power BI workspaces.

This exam is very hands-on. You need to know Power Query (M language), DAX (Data Analysis Expressions), and how to design effective visualisations. The April 2026 update adds Copilot features (creating visuals with Copilot, narrative visuals) and visual calculations using DAX.

**💡 Tip:** The three core domains (Prepare, Model, Visualise) each carry 25–30% — they're equally weighted. Don't neglect any of them. DAX is the most commonly failed area, so invest extra time there.

## Skills Measured — with Microsoft Learn Links

### Prepare the data (25–30%)

Data preparation is the first step in any Power BI project — connecting to data sources, profiling the data to understand its quality, and transforming it into a usable shape. Power Query is the tool you'll use for most of this work.

#### Get or connect to data

You can connect to hundreds of data sources — SQL databases, Excel files, SharePoint lists, web APIs, and shared semantic models. You need to choose the right connection mode: Import (loads data into Power BI), DirectQuery (queries the source live), or DirectLake (Fabric-specific, best of both).

- [Identify and connect to data sources or a shared semantic model](https://learn.microsoft.com/en-us/training/modules/get-data-power-bi/2-data-sources)
- [Change data source settings, including credentials and privacy levels](https://learn.microsoft.com/en-us/training/modules/get-data-power-bi/5-data-source-settings)
- [Choose between DirectLake, DirectQuery, and Import](https://learn.microsoft.com/en-us/training/modules/get-data-power-bi/3-storage-modes)
- [Create and modify parameters](https://learn.microsoft.com/en-us/training/modules/get-data-power-bi/6-parameters)

#### Profile and clean the data

Before transforming data, you need to understand it. Power Query provides column profiling, data statistics, and distribution analysis. You'll fix common issues like inconsistent values, nulls, data type mismatches, and import errors.

- [Evaluate data, including data statistics and column properties](https://learn.microsoft.com/en-us/training/modules/clean-data-power-bi/2-data-statistics)
- [Resolve inconsistencies, unexpected or null values, and data quality issues](https://learn.microsoft.com/en-us/training/modules/clean-data-power-bi/3-resolve-inconsistencies)
- [Resolve data import errors](https://learn.microsoft.com/en-us/training/modules/clean-data-power-bi/4-import-errors)

#### Transform and load the data

Transformation is where you shape data into the format your model needs — changing data types, creating calculated columns, grouping rows, pivoting/unpivoting, merging tables, and setting up proper keys for relationships.

- [Select appropriate column data types](https://learn.microsoft.com/en-us/training/modules/clean-data-power-bi/5-advanced-editor)
- [Create and transform columns](https://learn.microsoft.com/en-us/training/modules/clean-data-power-bi/5-advanced-editor)
- [Group and aggregate rows](https://learn.microsoft.com/en-us/power-query/group-by)
- [Pivot, unpivot, and transpose data](https://learn.microsoft.com/en-us/power-query/unpivot-column)
- [Convert semi-structured data to a table](https://learn.microsoft.com/en-us/power-query/parse-json-xml)
- [Create fact tables and dimension tables](https://learn.microsoft.com/en-us/training/modules/design-model-power-bi/2-tables)
- [Identify when to use reference or duplicate queries and the resulting impact](https://learn.microsoft.com/en-us/power-query/best-practices)
- [Merge and append queries](https://learn.microsoft.com/en-us/training/modules/clean-data-power-bi/6-combine-queries)
- [Identify and create appropriate keys for relationships](https://learn.microsoft.com/en-us/training/modules/design-model-power-bi/3-relationships)
- [Configure data loading for queries](https://learn.microsoft.com/en-us/training/modules/get-data-power-bi/4-load-data)

### Model the data (25–30%)

Data modelling is the heart of Power BI. A well-designed model makes reports fast and DAX formulas simple. A poorly designed model makes everything painful. Star schema (fact tables + dimension tables) is the recommended approach.

#### Design and implement a data model

You need to know how to configure table and column properties, create relationships with the right cardinality and cross-filter direction, implement role-playing dimensions (e.g., a single Date table used for both Order Date and Ship Date), and create a common date table.

- [Configure table and column properties](https://learn.microsoft.com/en-us/training/modules/design-model-power-bi/2-tables)
- [Implement role-playing dimensions](https://learn.microsoft.com/en-us/training/modules/design-model-power-bi/4-role-playing-dimensions)
- [Define a relationship's cardinality and cross-filter direction](https://learn.microsoft.com/en-us/training/modules/design-model-power-bi/3-relationships)
- [Create a common date table](https://learn.microsoft.com/en-us/training/modules/design-model-power-bi/5-date-table)
- [Identify use cases for calculated columns and calculated tables](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-add-calculated-tables/2-calculated-columns)

#### Create model calculations by using DAX

DAX (Data Analysis Expressions) is the formula language of Power BI. You need to know how to write measures (aggregations that respond to filter context), use CALCULATE (the most important DAX function), implement time intelligence (YTD, MTD, previous year comparisons), and create calculation groups.

- [Create single aggregation measures](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-add-measures/2-simple-measures)
- [Use the CALCULATE function](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-modify-filter/2-calculate-function)
- [Implement time intelligence measures](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-time-intelligence/2-functions)
- [Use basic statistical functions](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-add-measures/3-statistical-functions)
- [Create semi-additive measures](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-add-measures/4-semi-additive)
- [Create a measure by using quick measures](https://learn.microsoft.com/en-us/power-bi/transform-model/desktop-quick-measures)
- [Create calculated tables or columns](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-add-calculated-tables/2-calculated-columns)
- [Create calculation groups](https://learn.microsoft.com/en-us/training/modules/dax-power-bi-add-calculated-tables/3-calculation-groups)

#### Optimise model performance

A slow report is a bad report. Performance Analyzer shows you which visuals are slow, DAX Query View helps identify expensive measures, and reducing table granularity (fewer rows, fewer columns) improves both speed and file size.

- [Improve performance by identifying and removing unnecessary rows and columns](https://learn.microsoft.com/en-us/training/modules/optimize-model-power-bi/2-performance)
- [Identify poorly performing measures, relationships, and visuals by using Performance Analyzer and DAX query view](https://learn.microsoft.com/en-us/training/modules/optimize-model-power-bi/3-performance-analyzer)
- [Improve performance by reducing granularity](https://learn.microsoft.com/en-us/training/modules/optimize-model-power-bi/4-reduce-cardinality)
