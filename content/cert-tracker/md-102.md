## Who is this exam for?

The MD-102 is for endpoint administrators — the people who manage devices (Windows, iOS, Android, macOS) at scale using Microsoft Intune. If you deploy laptops with Autopilot, push apps through Intune, configure compliance policies, and manage Windows updates, this exam covers your daily work.

You should have experience with Intune, Microsoft Entra ID, Windows Autopilot, Defender for Endpoint, and Windows 365. This is a hands-on, practical exam — expect scenario-based questions about device enrollment, configuration profiles, and compliance policies.

This exam was updated on **April 28, 2026** and now includes Intune Suite add-ons (Endpoint Privilege Management, Remote Help, Cloud PKI, Advanced Analytics), Windows 365 Cloud PC deployment, and Security Copilot integration. It also counts toward the **Microsoft 365 Certified: Administrator Expert** certification.

## Skills Measured — with Microsoft Learn Links

### Prepare infrastructure for devices (25–30%)

This domain covers getting devices into your management environment — joining them to Entra ID, enrolling them in Intune, and setting up compliance and identity requirements. You need to know the difference between Entra join, hybrid join, and registration, and how enrollment works across platforms.

#### Add devices to Microsoft Entra ID

Devices can connect to Entra ID in three ways: Entra join (cloud-only), Entra hybrid join (synced with on-premises AD), and Entra registration (BYOD). Each has different capabilities and use cases. You also need to know how to plan device groups for targeting policies.

- [Choose an appropriate device join type](https://learn.microsoft.com/en-us/entra/identity/devices/overview)
- [Join devices to Microsoft Entra ID](https://learn.microsoft.com/en-us/entra/identity/devices/how-to-hybrid-join)
- [Register devices to Microsoft Entra ID](https://learn.microsoft.com/en-us/entra/identity/devices/concept-device-registration)
- [Plan and implement groups for devices in Microsoft Entra ID](https://learn.microsoft.com/en-us/entra/identity/devices/manage-device-identities)

#### Enroll devices to Microsoft Intune

Enrollment brings devices under Intune management. Automatic enrollment works for Windows devices joined to Entra ID. iOS, Android, and macOS devices have their own enrollment methods. You need to know how to configure enrollment profiles for different Android deployment scenarios (fully managed, dedicated, work profile).

- [Configure enrollment settings](https://learn.microsoft.com/en-us/mem/intune/enrollment/enrollment-restrictions-set)
- [Configure automatic enrollment for Windows and bulk enrollment for iOS/iPadOS and Android](https://learn.microsoft.com/en-us/mem/intune/enrollment/windows-enroll)
- [Configure enrollment profiles for Android devices](https://learn.microsoft.com/en-us/mem/intune/enrollment/android-fully-managed-enroll)

#### Implement identity and compliance

Compliance policies define the minimum requirements for a device to be considered "healthy" (e.g., must have a PIN, must be encrypted, must not be jailbroken). Conditional Access policies can then block non-compliant devices from accessing corporate resources. This section also covers LAPS and managing local admin groups.

- [Manage roles in Intune](https://learn.microsoft.com/en-us/mem/intune/fundamentals/role-based-access-control)
- [Implement compliance policies for all supported device platforms](https://learn.microsoft.com/en-us/mem/intune/protect/device-compliance-get-started)
- [Implement Entra Conditional Access policies that require compliance status](https://learn.microsoft.com/en-us/mem/intune/protect/conditional-access-intune-common-ways-use)
- [Configure Windows Hello for Business](https://learn.microsoft.com/en-us/mem/intune/protect/identity-protection-windows-settings)
- [Implement and manage Windows LAPS](https://learn.microsoft.com/en-us/mem/intune/protect/windows-laps-overview)
- [Manage the membership of local groups on Windows devices](https://learn.microsoft.com/en-us/mem/intune/protect/intune-account-protection-profile-settings)

### Manage and maintain devices (30–35%)

This is the largest domain. It covers deploying Windows using Autopilot and provisioning packages, creating configuration profiles for all platforms, using Intune Suite add-ons, performing remote actions, and deploying Windows 365 Cloud PCs. Heavy hands-on content here.

#### Deploy and upgrade Windows clients by using cloud-based tools

Windows Autopilot enables zero-touch deployment — a new device boots up, connects to the internet, and automatically configures itself with apps, settings, and policies. You need to know the different Autopilot deployment modes (user-driven, self-deploying, pre-provisioned) and how to create an Enrollment Status Page.

- [Choose between Windows Autopilot and provisioning packages](https://learn.microsoft.com/en-us/autopilot/windows-autopilot)
- [Choose a Windows Autopilot deployment mode](https://learn.microsoft.com/en-us/autopilot/tutorial/autopilot-scenarios)
- [Apply a device name template](https://learn.microsoft.com/en-us/autopilot/profiles)
- [Implement Windows client deployment by using Windows Autopilot](https://learn.microsoft.com/en-us/autopilot/tutorial/user-driven/entra-join-workflow)
- [Create an Enrollment Status Page (ESP)](https://learn.microsoft.com/en-us/mem/intune/enrollment/windows-enrollment-status)
- [Plan and implement provisioning packages](https://learn.microsoft.com/en-us/windows/configuration/provisioning-packages/provisioning-packages)
- [Plan and implement device upgrades for Windows 11](https://learn.microsoft.com/en-us/mem/intune/protect/windows-10-feature-updates)
- [Implement a Windows 365 Cloud PC deployment](https://learn.microsoft.com/en-us/windows-365/enterprise/overview)

#### Plan and implement device configuration profiles

Configuration profiles are how you push settings to devices — Wi-Fi, VPN, email, certificates, restrictions, and more. Intune supports profiles for Windows, Android, iOS/iPadOS, macOS, and Windows 11 Enterprise multi-session (for AVD). Filters let you target profiles to specific subsets of devices.

- [Create device configuration profiles for Windows devices, including importing ADMX files](https://learn.microsoft.com/en-us/mem/intune/configuration/device-profile-create)
- [Create device configuration profiles for Android devices](https://learn.microsoft.com/en-us/mem/intune/configuration/device-restrictions-android-for-work)
- [Create device configuration profiles for iOS/iPadOS devices](https://learn.microsoft.com/en-us/mem/intune/configuration/device-restrictions-ios)
- [Create device configuration profiles for macOS devices](https://learn.microsoft.com/en-us/mem/intune/configuration/device-restrictions-macos)
- [Create device configuration profiles for Windows 11 Enterprise multi-session devices](https://learn.microsoft.com/en-us/mem/intune/fundamentals/azure-virtual-desktop-multi-session)
- [Target a profile by using filters](https://learn.microsoft.com/en-us/mem/intune/fundamentals/filters)

#### Implement Intune Suite add-on capabilities

The Intune Suite adds premium features beyond the base Intune licence. Endpoint Privilege Management lets standard users elevate specific apps without giving them full admin access. Remote Help provides helpdesk-style screen sharing. Cloud PKI issues certificates without on-premises infrastructure.

- [Configure Endpoint Privilege Management](https://learn.microsoft.com/en-us/mem/intune/protect/epm-overview)
- [Manage applications by using the Enterprise App Catalog](https://learn.microsoft.com/en-us/mem/intune/apps/apps-enterprise-app-management)
- [Implement Microsoft Intune Advanced Analytics](https://learn.microsoft.com/en-us/mem/intune/fundamentals/advanced-endpoint-analytics)
- [Configure Microsoft Intune Remote Help](https://learn.microsoft.com/en-us/mem/intune/fundamentals/remote-help)
- [Identify use cases for Microsoft Cloud PKI](https://learn.microsoft.com/en-us/mem/intune/protect/microsoft-cloud-pki-overview)
- [Implement Microsoft Tunnel for Mobile Application Management](https://learn.microsoft.com/en-us/mem/intune/protect/microsoft-tunnel-mam)

#### Perform remote actions on devices

Remote actions let you manage devices without physical access — sync policies, restart, retire (remove corporate data), or full wipe (factory reset). Bulk actions let you do this across hundreds of devices. Device query using KQL lets you run real-time queries against device inventory.

- [Sync, restart, retire, or wipe devices](https://learn.microsoft.com/en-us/mem/intune/remote-actions/device-management)
- [Perform bulk remote actions](https://learn.microsoft.com/en-us/mem/intune/remote-actions/bulk-device-actions)
- [Update Microsoft Defender Antivirus security intelligence](https://learn.microsoft.com/en-us/mem/intune/protect/endpoint-security-antivirus-policy)
- [Rotate BitLocker recovery keys](https://learn.microsoft.com/en-us/mem/intune/protect/encrypt-devices)
- [Run a device query by using KQL](https://learn.microsoft.com/en-us/mem/intune/fundamentals/device-query-with-kusto)

### Manage applications (15–20%)

This domain covers deploying apps to managed devices (Win32 apps, MSI, Microsoft 365 Apps, store apps) and protecting corporate data within apps using app protection policies. App protection is especially important for BYOD scenarios where you don't manage the device itself.

#### Deploy and update apps

- [Prepare applications for deployment by using Intune](https://learn.microsoft.com/en-us/mem/intune/apps/apps-add)
- [Deploy apps by using Intune](https://learn.microsoft.com/en-us/mem/intune/apps/apps-deploy)
- [Deploy Microsoft 365 Apps by using Intune](https://learn.microsoft.com/en-us/mem/intune/apps/apps-add-office365)
- [Configure policies for Office apps](https://learn.microsoft.com/en-us/mem/intune/apps/app-office-policies)
- [Deploy Microsoft 365 Apps as part of a Windows Autopilot deployment](https://learn.microsoft.com/en-us/mem/intune/apps/apps-add-office365)
- [Manage Microsoft 365 Apps by using the Microsoft 365 Apps admin center](https://learn.microsoft.com/en-us/deployoffice/admincenter/overview)
- [Deploy apps from platform-specific app stores by using Intune](https://learn.microsoft.com/en-us/mem/intune/apps/store-apps-microsoft)

#### Plan and implement app protection and app configuration policies

App protection policies (APP, formerly MAM) protect corporate data within apps — even on unmanaged devices. For example, you can prevent users from copying data from Outlook to a personal app, or require a PIN to open a managed app. App configuration policies push settings to apps (like pre-configuring an email account).

- [Plan and implement app protection policies](https://learn.microsoft.com/en-us/mem/intune/apps/app-protection-policy)
- [Implement Entra Conditional Access policies for app protection policies](https://learn.microsoft.com/en-us/mem/intune/protect/app-based-conditional-access-intune)
- [Plan and implement app configuration policies for managed apps and managed devices](https://learn.microsoft.com/en-us/mem/intune/apps/app-configuration-policies-overview)

### Protect devices (15–20%)

This domain covers endpoint security — antivirus, disk encryption, firewalls, attack surface reduction, security baselines, and Defender for Endpoint integration. It also covers managing Windows updates through Intune (update rings, feature updates, driver updates) and handling updates for iOS, Android, and macOS.

#### Configure endpoint security

Security baselines are pre-configured sets of Windows settings recommended by Microsoft's security team. Applying them gives you a solid security foundation without configuring dozens of individual settings. You also need to know how to create custom antivirus, encryption, and firewall policies.

- [Create antivirus policies](https://learn.microsoft.com/en-us/mem/intune/protect/endpoint-security-antivirus-policy)
- [Create disk encryption policies](https://learn.microsoft.com/en-us/mem/intune/protect/encrypt-devices)
- [Create firewall policies](https://learn.microsoft.com/en-us/mem/intune/protect/endpoint-security-firewall-policy)
- [Configure Attack surface reduction policies](https://learn.microsoft.com/en-us/mem/intune/protect/endpoint-security-asr-policy)
- [Plan and implement security baselines](https://learn.microsoft.com/en-us/mem/intune/protect/security-baselines)
- [Integrate Intune with Microsoft Defender for Endpoint](https://learn.microsoft.com/en-us/mem/intune/protect/advanced-threat-protection-configure)
- [Onboard devices into Microsoft Defender for Endpoint](https://learn.microsoft.com/en-us/defender-endpoint/onboarding)

#### Manage device updates by using Intune

Windows update management in Intune uses update rings (control quality and feature update timing), update policies (target specific feature updates), and Delivery Optimization (peer-to-peer download to reduce bandwidth). For iOS, macOS, and Android, updates are managed differently through configuration profiles or FOTA.

- [Plan for device updates](https://learn.microsoft.com/en-us/mem/intune/protect/windows-update-for-business-configure)
- [Create and manage update rings by using Intune](https://learn.microsoft.com/en-us/mem/intune/protect/windows-10-update-rings)
- [Create and manage update policies by using Intune, including iOS/iPadOS and macOS](https://learn.microsoft.com/en-us/mem/intune/protect/software-updates-ios)
- [Manage Android updates by using configuration profiles or FOTA deployments](https://learn.microsoft.com/en-us/mem/intune/protect/android-os-update-management)
- [Configure Windows client Delivery Optimization by using Intune](https://learn.microsoft.com/en-us/mem/intune/configuration/delivery-optimization-settings)
- [Monitor updates](https://learn.microsoft.com/en-us/mem/intune/protect/windows-update-reports)
