## About the SCAZT Exam

> Master Cisco Secure Access — Zero Trust Network Access, SSE/SASE, identity and device trust, security policy, and cloud-delivered operations

The 300-740 SCAZT (Designing and Implementing Secure Cloud Access for Users and Endpoints) is a CCNP Security concentration exam built on the Cisco Secure Access Security Service Edge (SSE/SASE) platform. It validates your ability to design and implement secure cloud access: the Secure Access architecture and Zero Trust Network Access (ZTNA) concepts, identity-provider integration and multi-factor authentication with Cisco Duo, device posture and trust, security policy (web, DNS-layer, cloud-delivered firewall, DLP, malware, and threat protection), the access models (Secure Internet Access, Secure Private Access / ZTNA, and VPN as a Service), resource connectors and tunnels, and cloud-delivered operations and troubleshooting. Passing SCAZT earns the Cisco Certified Specialist - Secure Cloud Access badge and, with the SCOR 350-701 core, the CCNP Security certification. Original practice questions. Not affiliated with, endorsed by, or sourced from Cisco Systems certification exams.

## Who Should Take This Exam?

SCAZT is designed for **security engineers, cloud security architects, and SASE/SSE administrators who design and operate Cisco Secure Access**. Cisco recommends 3-5 years of security-solution implementation experience; hands-on time with Cisco Secure Access, Duo, and Zero Trust access policy helps a lot.

**Typical study time:** 6-10 weeks of focused study

## Exam Domains & Weights

The SCAZT exam covers **5 domains**. Focus your study time based on the weights below — higher-weighted domains have more exam questions.

| Domain | Weight | Practice Qs |
|--------|--------|-------------|
| Cisco Secure Access Concepts | 10% | 25 |

| Policies | 30% | 75 |
| Access | 30% | 75 |
| Operations | 10% | 25 |
| **Total** | **100%** | **250** |

> 💡 **Study tip:** **Policies** and **Access** each carry 30% — together that is 60% of the exam, so start there. Nail the boundaries that Cisco loves to test: **SSE** (SWG + CASB + ZTNA + FWaaS) is the security half of **SASE** (which adds SD-WAN); **ZTNA** grants per-application least-privilege access while **VPN as a Service** gives broader network-style reach; and **DNS-layer security** blocks at resolution while a **Secure Web Gateway** does full proxy inspection (which needs TLS decryption for HTTPS).
