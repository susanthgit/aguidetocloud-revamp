## About the GH-100 Exam

> Master GitHub Enterprise administration — identities and SSO/SCIM, access and permissions at scale, deployment and licensing, security and compliance governance, Actions administration, and Packages management.

The complete practice exam for GitHub Administration (GH-100), the Administering GitHub Enterprise certification. Covers supporting GitHub Enterprise for users and stakeholders (support bundles, audit logs, developer-process standards), managing identities and authentication (Enterprise Managed Users, SAML SSO, SCIM provisioning, team synchronization, 2FA enforcement), GitHub deployment and licensing (Enterprise Cloud vs Server, data residency, seat consumption), access and permissions (enterprise, organization, and repository roles, teams and nested teams, rulesets, least-privilege design), secure development and compliance (secret scanning and push protection, code scanning, Dependabot, audit log streaming, personal access token and GitHub App policies), administering GitHub Actions (runner groups, hosted and self-hosted runners, allowed actions, organization and repository secrets, OIDC), and managing GitHub Packages (registries, visibility, granular permissions) — every question a real-world scenario with full explanations.

## Who Should Take This Exam?

The GH-100 is designed for **GitHub Enterprise administrators, DevOps engineers, and technology managers** who govern GitHub at organization and enterprise scale. Intermediate hands-on experience administering GitHub Enterprise is recommended.

**Prerequisites:** None (intermediate GitHub Enterprise administration experience recommended)

**Typical study time:** 4-8 weeks of focused study

## Exam Domains & Weights

The GH-100 exam covers **7 domains**. Focus your study time based on the weights below — higher-weighted domains have more exam questions.

| Domain | Weight | Practice Qs |
|--------|--------|-------------|
| Support GitHub Enterprise for users and key stakeholders | 15% | 36 |
| Manage user identities and GitHub authentication | 20% | 48 |
| Describe how GitHub is deployed, distributed, and licensed | 5% | 17 |
| Manage access and permissions based on membership | 20% | 48 |
| Enable secure software development and ensure compliance | 15% | 36 |
| Manage GitHub Actions | 20% | 48 |
| Manage GitHub Packages | 5% | 17 |
| **Total** | **100%** | **250** |

> 💡 **Study tip:** Three domains carry 20% each — **identities and authentication**, **access and permissions**, and **GitHub Actions** — so they account for most of the exam. Start there. The two 5% domains (**deployment and licensing**, **Packages**) are smaller, but don't skip them: a handful of questions can still tip a borderline pass.
