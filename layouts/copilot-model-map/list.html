What can you actually control as an M365 admin? Here's the truth — most model routing is automatic, but there are key levers you can pull.

Full Control

Copilot Studio — Model Selection

Agent creators can explicitly choose which AI model powers their custom agent. Options include OpenAI GPT-5.x, Claude Opus, Claude Sonnet, and GPT-4o.

How: Copilot Studio → Agent settings → Model selection
Microsoft Learn docs →
Limited Control

Multi-Model Opt-In

Tenant admins can enable or disable Anthropic Claude models for Copilot Chat and Researcher. When enabled, Copilot can auto-route to Claude for suitable tasks.

How: Microsoft 365 admin centre → Copilot settings → Model providers
Announcement blog →
Limited Control

Frontier Programme

Opt-in programme for advanced multi-model features like Researcher's Critique and Council modes. Requires admin approval.

How: Microsoft 365 admin centre → Copilot → Frontier settings
Limited Control

In-App Copilot (Word, Excel, PowerPoint, etc.)

Microsoft manages per-request model routing automatically. However, admins control whether Anthropic models are enabled at tenant level — which affects Agent Mode and advanced features in Office apps.

What you control: Anthropic provider enablement, user/group scoping. What you don't: Per-request model selection within a feature.
Limited Control

Cowork & Analyst

These built-in agents use fixed model assignments (Claude for Cowork, o3-mini for Analyst). Admins can control availability — deploy Cowork to all users, specific groups, or block it entirely.

What you control: Feature availability and user/group scoping. What you don't: Which model the agent uses internally.
