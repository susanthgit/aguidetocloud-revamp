{{/* ═══ TAB: OPENAI VS ANTHROPIC ═══ */}}

When Copilot uses OpenAI models, your data stays inside Microsoft's boundary. When it uses Anthropic Claude, data crosses to a sub-processor. Here's exactly what's different.

OpenAI Models (GPT)

Default
Hosted byMicrosoft (Azure OpenAI)
Data boundaryInside Microsoft 365 service boundary
EU Data Boundary✅ Supported
Data residencyYour tenant's region
Training on your data❌ Never
Contractual coverMicrosoft Product Terms + DPA
Admin opt-in neededNo — enabled by default
Data retentionTransient processing only. No persistent LLM storage.

Anthropic Models (Claude)

Sub-processor
Hosted byAnthropic (under Microsoft contractual control)
Data boundaryCrosses Microsoft boundary → Anthropic infrastructure
EU Data Boundary❌ Excluded — disabled by default in EU/EFTA/UK
Data residencyProcessed in US-based infrastructure
Training on your data❌ Never — covered by Microsoft DPA
Contractual coverMicrosoft Product Terms + DPA (not Anthropic's own terms)
Admin opt-in neededYes (disabled by default in EU/UK, enabled elsewhere)
Data retentionTransient processing only. No persistent storage at Anthropic.
Key takeaway: Both providers are contractually bound by Microsoft's DPA. Neither trains on your data. The difference is WHERE processing happens — OpenAI stays inside Microsoft, Anthropic crosses to a sub-processor.
